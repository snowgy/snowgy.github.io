Algebricks: A Data Model-Agnostic Compiler

Posted on 2019-07-07 | In research |

Algebricks是一个query语言的后端编译器。他的精妙之处在于抽象。现有的query compiler基本都是依赖于data-model的，关系型非关系型数据库的编译器要用两种编译器。但是Algebricks这个框架把language-sepcific的部分和data model-dependent的部分和通用的部分分开了。That’s why it can be used to develop multiple languages with different data models.

Algebricks: A Data Model-Agnostic Compiler Backend for Big Data Languages

Motivation

Most of the query compilers these days are data model-dependent. In this work, they describe a new query compiler achitecture that seperates language-specific and data model-dependent aspects from a more general query compiler backend.

Algebricks can generate executable data-parallel programs for shared nothing clusters and can be used to develop multiple languages with different data models.

The output of Algebricks are jobs that can be scheduled and executed in parallel on the Hyracks runtime platform.

The Hyracks Runtime

Two bottom layers : the general-purpose DAG execution engine and its associated runtime libraries.

Execution Engine

Hyracks is a push-based data-parallel runtime.

Jobs are submitted to Hyracks in the form of DAGs made up of operators and connectors.

Compilation Flow

  1. An incoming query string is first lexically analyzed and parsed to construct an abstract syntax tree (AST).
  2. This AST is then translated into the Algrebricks query plan.
  3. The logical plan is then handed to the logical optimizer which rewrites it heuristically using logical rewrite rules.
  4. The physical optimizer translates the optimized logical plan into an Algebricks physical plan by selecting physical operators

Logical Operators vs Physical Operators

While the logic plan operators capture the semantics of an Algebricks logical plan, physical operators are used to specify the exact algorithm to use for evaluating each operator. During the query rewrite process rules assign physical operators to each logical operator in the query plan.

Most logic operators in Algebricks map to a unique physical operator except Join and group by operators.

Algebricks also provides physical operators to enforce order properties as well as partitioning properties of data distributed to individual operations during parallel evaluation of the query.
